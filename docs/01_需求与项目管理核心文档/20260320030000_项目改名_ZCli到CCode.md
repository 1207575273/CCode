# 项目改名：ZCli → CCode

> 日期: 2026-03-20
> 状态: 已完成
> 影响范围: GitHub 仓库、npm 包名、CLI 命令名、配置目录、指令文件、环境变量、README

---

## 一、背景

### 1.1 原始命名

项目创建时命名为 **ZCli**（GitHub 仓库名 **ZCode**），取 Z 作为通用前缀。随着项目推进到 v0.0.1 里程碑，准备发布到 npm 时遇到了命名冲突问题。

### 1.2 改名动机

| 问题 | 详情 |
|------|------|
| **GitHub 重名** | `ZCode` 是一个常见的仓库名，GitHub 上有多个同名项目，搜索辨识度低 |
| **npm 冲突** | `zcli` 包名被 Zendesk 占用（`@zendesk/zcli-themes`），`zcli-ai` 虽可用但辨识度不够 |
| **品牌含义缺失** | "Z" 没有明确的含义，无法传达项目的身份和定位 |

### 1.3 新名称：CCode

**CCode** 的 **C** 承载三层含义：

- **C**odeYang — 作者英文名，个人品牌标识
- **C**hina — 中国开发者出品的 AI 编程工具
- **C**ode Agent — 代码智能体，直点核心能力

新名称简洁、好记、不与已有项目冲突，且与 "Claude Code" 形成自然的品牌联想（同为 Code Agent 工具）。

## 三、改造过程

### 3.1 GitHub 仓库改名

在 GitHub 仓库 Settings → Repository name 中将 `ZCode` 改为 `CCode`。

GitHub 自动为旧 URL 设置 301 重定向（有效期约 90 天）。

### 3.2 本地 Git Remote 更新

```bash
git remote set-url origin git@github.com:1207575273/CCode.git
```

### 3.3 package.json 更新

```json
{
  "name": "ccode-cli",                          // ← 从 zcli-ai 改
  "bin": { "ccode": "./dist/bin/ccode.js" },    // ← 命令名 + 产物名
  "repository": "git+https://github.com/1207575273/CCode.git",
  "homepage": "https://github.com/1207575273/CCode#readme"
}
```

### 3.4 全局 `.zcli` → `.ccode` 替换

使用 `find + sed` 批量替换 `src/` 和 `tests/` 下所有 `.ts` 文件：

```bash
# 第一轮：目录和文件路径
find src/ tests/ -name "*.ts" -exec sed -i 's/\.zcli/\.ccode/g' {} +

# 第二轮：环境变量和内部变量
find src/ tests/ -name "*.ts" -exec sed -i \
  -e 's/ZCLI_HOME/CCODE_HOME/g' \
  -e 's/ZCLI_CWD/CCODE_CWD/g' \
  -e 's/ZCLI_TRIGGER/CCODE_TRIGGER/g' \
  -e 's/ZCLI_TOOL_NAME/CCODE_TOOL_NAME/g' \
  -e 's/ZCLI_TOOL_CALL_ID/CCODE_TOOL_CALL_ID/g' \
  -e 's/ZCLI_PLUGIN_ROOT/CCODE_PLUGIN_ROOT/g' \
  {} +

# 第三轮：指令文件名
find src/ tests/ -name "*.ts" -exec sed -i 's/ZCLI\.md/CCODE\.md/g' {} +

# 第四轮：入口文件引用
mv bin/zcli.ts bin/ccode.ts
sed -i "s/entry: \['bin\/zcli.ts'\]/entry: ['bin\/ccode.ts']/" tsup.config.ts
```

### 3.5 README 全面更新

- 标题：`# ZCli` → `# CCode`
- 新增改名说明段落（含原因、新名寓意、迁移指南）

## 五、用户迁移指南

### 从旧版 ZCli 迁移到 CCode

如果你之前使用过 ZCli（`.zcli/` 目录下有配置），只需一步：

```bash
# Linux / macOS
mv ~/.zcli ~/.ccode

# Windows
ren "%USERPROFILE%\.zcli" ".ccode"
```

配置文件内容完全兼容，无需修改 `config.json`、`.mcp.json` 等文件内容。

### 项目级目录迁移

```bash
# 每个使用 ZCli 的项目根目录下
mv .zcli .ccode
```

同时将指令文件改名：
- `ZCLI.md` → `CCODE.md`（可选，`CLAUDE.md` 仍然兼容）

### 新用户

直接使用，首次启动自动创建 `~/.ccode/` 目录和默认配置。

```bash
# npm 安装
npm install -g ccode-cli
ccode

# 或 npx 临时使用
npx ccode-cli
```
