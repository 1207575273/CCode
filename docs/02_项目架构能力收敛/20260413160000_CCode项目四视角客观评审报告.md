## 三、关键发现（评分依据）

### 3.1 架构亮点

1. **AsyncGenerator 事件流引擎**
   - `AgentLoop.run()` yield 事件（text / tool_start / tool_done / error / done）
   - UI 层和 SessionLogger 独立消费，观察者模式解耦
   - 学术参考价值高

2. **三层抽象设计**
   - **Tool 层**：Tool / StreamableTool 双接口，Registry 支持 cloneWith/cloneWithout 权限隔离
   - **Provider 层**：协议归一化（Anthropic ↔ OpenAI 消息格式转换），Session 级实例池化
   - **Memory 层**：混合检索（BM25 关键词 + 向量语义），jieba 中文分词

3. **SubAgent 编排**
   - 独立模型/Provider 选择
   - JSONL 会话隔离（virtualSessionId）
   - 父上下文快照传递
   - 多来源停止控制（CLI/Web/Parent/Timeout）

4. **JSONL 持久化设计**
   - Append-only 事件流，零事务开销
   - parentUuid 支持会话分支
   - 子 Agent 独立文件隔离

5. **可扩展性**
   - MCP 协议集成（stdio/SSE/streamable-http）
   - Skills 引擎（解析/加载/内置）
   - Runtime Plugins 系统
   - Hooks 事件钩子

### 3.2 关键缺陷

#### 工程质量

| 问题 | 位置 | 严重度 | 影响 |

## 五、各视角核心结论

### 5.1 大学教授视角（70.6分）

**值得学习的设计**：
- AsyncGenerator 事件流模式实现了真正的关注点分离，UI/Logger/Persistence 各自独立消费
- 混合 RAG 检索（BM25 + 向量）兼顾精确匹配和语义理解
- Tool Registry 的 cloneWith/cloneWithout 实现了优雅的权限隔离
- JSONL + parentUuid 的会话分支模型比传统数据库方案更轻量

**需要改进的地方**：
- agent-loop.ts 认知复杂度过高（55个控制流语句），应拆分为子策略
- O(n²) 分支发现算法应优化为 O(n) 的 Map 查找
- 缺乏形式化的架构文档和算法复杂度分析

### 5.2 工程师视角（66.6分）— 最低分

**做得好的**：
- 核心模块测试覆盖扎实（tools/core/memory/mcp）
- TypeScript strict 全家桶 + Zod 运行时验证
- 指数退避 retry + AbortSignal 取消传播
- Mock 工厂模式专业

**必须改进的**：
- 27个空 catch 块 → 至少加日志或注释说明为什么忽略
- 无 CI/CD → 最低限度：GitHub Actions 跑 `npm test` + 覆盖率报告
- UI 0% 测试 → ChatPage / InputBar / PermissionCard 至少覆盖关键路径
- useChat.ts 870行 → 拆分为 useAgentLoop / useToolEvents / useSubAgents 等子 Hook
- WebSocket silent failure → 关键事件（permission_request, error）必须有重试或日志

### 5.3 架构师视角（73.5分）— 最高分

**架构优势**：
- 15个功能模块边界清晰，高内聚低耦合
- Event-driven 架构天然支持录制/回放/观测
- Provider 协议归一化屏蔽了上游差异
- 插件化设计（MCP/Skills/Plugins/Hooks）预留了充分的扩展空间

**架构风险**：
- Tool Registry 初始化后仍可变，缺少 freeze 机制
- JSONL 无并发写保护，多线程场景有隐患
- MCP 工具名冲突未检测，两个 Server 导出同名工具会静默覆盖
- WebSocket 无反压（backpressure），慢客户端可能阻塞 CLI

### 5.4 产品视角（70.6分）

**竞争优势**：
- 中国市场唯一开源多模型 CLI Agent（GLM/DeepSeek/Ollama 原生支持）
- jieba 中文分词在 Memory/RAG 场景是独特优势
- Web Dashboard 可视化是对标 Claude Code 的差异化功能
- 配置示例丰富，README 对用户友好

**用户体验短板**：
- 缺 CHANGELOG，用户升级无从参考
- 静默错误丢失（WebSocket、JSONL）影响用户信任
- Beta 阶段粗糙边角明显（如 SubAgent 回放丢历史、stop 按钮断连等已修复问题说明质量关口不足）
