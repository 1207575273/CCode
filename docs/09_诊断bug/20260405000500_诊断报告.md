## 四、解决方案

### 问题 1：token usage 兼容性

- **方案 A**：检查 GLM API 文档，确认是否支持 `stream_options: { include_usage: true }`
- **方案 B**：在 stream 结束后，从 `response_metadata` 中提取（某些模型在这里放 usage）
- **方案 C**：加一个 fallback —— 如果 stream 结束后 usage 仍为空，用 invoke 调一次获取 token 计数（性能换正确性）

### 问题 2：RepetitionDetector 阈值优化

- **现状**：warn=3, block=5，实测中模型在 warn 后还是试了 1 次才切换
- **建议**：调整为 warn=2, block=4 —— 给模型 1 次重试机会（可能真的有网络抖动），但更快拦截
- **验证**：改完后再跑一次同样的任务
