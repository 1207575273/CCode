# ZCli Diff 预览功能实现记录

> 完成日期：2026-03-09 | 分支：main | 测试：91/91

---

## 一、需求背景

ZCli 的 agent loop 调用 `edit_file` / `write_file` 等危险工具时，PermissionDialog 仅显示工具名和文件路径，用户无法在审批前看到具体变更内容。需要在审批弹窗中嵌入 unified diff 预览，让用户看清改了什么再做决定。

---

## 二、技术选型

| 决策 | 选择 | 原因 |
|------|------|------|
| diff 库 | `diff` (jsdiff) v8.0.3 | npm 生态最成熟，零额外依赖，自带 TS 类型，`structuredPatch()` 直出 hunks |
| 显示样式 | Unified（单列） | 终端标准范式（git/Claude Code 同款），窄宽度可用 |
| 覆盖工具 | edit_file + write_file | bash 命令无法在执行前预知文件变更，投入产出比低 |
| 新建文件 | 前 20 行 + 截断提示 | 兼顾可见性和防刷屏 |

---

## 三、实现架构

```
PermissionDialog
  └─ useMemo: 根据 toolName 调用对应计算函数
       ├─ edit_file  → computeEditDiff(path, old_str, new_str)
       ├─ write_file → computeWriteDiff(path, content)
       └─ 其他工具   → null（保持原有预览）
  └─ diffData !== null → <DiffView />
     diffData === null → 原有纯文本预览
```

### 三层分离

| 层 | 文件 | 职责 |
|----|------|------|
| 计算层 | `src/utils/compute-diff.ts` | 读文件、算 diff、产出 DiffData 结构 |
| 展示层 | `src/ui/DiffView.tsx` | 纯展示组件，接收 DiffData 渲染 unified diff |
| 集成层 | `src/ui/PermissionDialog.tsx` | useMemo 调用计算层，条件渲染展示层 |

---

## 四、核心模块说明

### 4.1 compute-diff.ts

**导出函数：**

- `computeEditDiff(filePath, oldStr, newStr): DiffData` — 读原文件，替换后对比
- `computeWriteDiff(filePath, newContent): DiffData` — 读原文件（不存在则空串），对比新内容

**边界处理：**

| 场景 | 处理 |
|------|------|
| 文件不存在（edit） | `error: "File not found: ..."` |
| 二进制文件 | `error: "Binary file — diff not available"` |
| 新文件 > 20 行 | 截断至 20 行，`truncatedLines` 记录溢出行数 |
| 大 diff > 5 hunks | `buildHunks` 截取前 5 个 |
| 非 ENOENT 异常 | 向上抛出（非静默吞掉） |

### 4.2 DiffView.tsx

纯展示组件，无 state / useInput / 副作用。

- `+` 行：绿色
- `-` 行：红色
- 上下文行：dimColor
- Hunk 头：`@@ -old,lines +new,lines @@`
- 统计行：`+N -M`（仅 > 0 时显示）
- 新建文件：路径后追加绿色 `(new)` 标签
- 截断提示：`... 还有 N 行未显示`

### 4.3 PermissionDialog 变更

- 新增 `useMemo` 计算 diffData
- 标题逻辑：`isNewFile` 时显示 "Create file"
- 条件渲染：有 diffData → DiffView，无 → 原有 formatPreview
- useInput 键盘处理、OPTIONS 选项完全不变
