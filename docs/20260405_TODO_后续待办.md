  - 如果是 tsx → 改动应该已生效，问题在 GLM API 侧

- [ ] **确认 GLM API 支持情况**：开 debug 模式（`DEBUG=ccode:* npm run dev`），跑一次对话，grep 日志中的 `usage_metadata` 或 `stream_options`

  - 如果 GLM API 不支持 stream_options → 需要 fallback 方案

- [ ] **实现 fallback 方案**（如果 GLM 不支持 stream_options）：
  ```
  方案 A：从 response_metadata 中提取（某些模型在这里放 usage）
  方案 B：stream 结束后如果 usage 为空，从最后一个 chunk 的 usage_metadata 单独提取
  方案 C：在 openai-compat.ts 中，如果 stream 结束后 usageMeta 为 null，
          检查 allChunks 的最后一个 chunk 是否有 usage_metadata（不依赖 concat）
  ```
  
  **推荐方案 C**，改动最小：
  ```typescript
  // 在 allChunks.reduce 之后
  if (!usageMeta && allChunks.length > 0) {
    // 某些模型（GLM）的 usage 只在最后一个 chunk 上，concat 可能丢失
    const lastChunk = allChunks[allChunks.length - 1]
    usageMeta = lastChunk.usage_metadata ?? lastChunk.response_metadata?.usage ?? null
  }
  ```
