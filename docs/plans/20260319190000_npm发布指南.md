## 二、发布前需要确认/修改的项

### 1. 包名选择

当前包名 `zcli-ai`，npm 上未被占用。可选方案：

| 包名 | `npx` 用法 | 全局安装用法 | 备注 |
|------|-----------|-------------|------|
| `zcli-ai` | `npx zcli-ai` | `npm i -g zcli-ai && zcli` | 当前配置 |
| `zcode-cli` | `npx zcode-cli` | `npm i -g zcode-cli && zcli` | 与 GitHub 仓库名对齐 |
| `@codeyang/zcli` | `npx @codeyang/zcli` | `npm i -g @codeyang/zcli && zcli` | scope 包，不会重名 |

> **注意**：包名决定了 `npx <包名>` 的用法，但全局安装后执行的命令名由 `bin` 字段决定（当前是 `zcli`）。

### 2. GitHub 仓库名

当前仓库名：`ZCode`（https://github.com/1207575273/ZCode）

如果要改名，在 GitHub → Settings → Repository name 修改。改名后：
- 旧 URL 自动 301 重定向（一段时间内有效）
- 本地需要更新 remote：
  ```bash
  git remote set-url origin git@github.com:1207575273/新名字.git
  ```
- package.json 的 `repository` 和 `homepage` 字段也要同步更新

### 3. bin 命令名

当前 `bin.zcli`，全局安装后用户执行 `zcli`。如果想改成别的：

```json
"bin": {
  "zcode": "./dist/bin/zcli.js"    ← 命令名改为 zcode
}
```

### 4. 版本号

当前 `0.0.1`。npm 语义版本规范：
- `0.0.x` — 初始开发，随时可能有破坏性变更
- `0.1.0` — 有了基本可用的功能集
- `1.0.0` — 正式发布

## 三、发布流程

### 前置条件

```bash
# 确认已登录 npm
npm whoami
# 应输出：yang1207575273
```

### 步骤

```bash
cd zCli

# 1. 确认版本号（修改 package.json 中的 version 如需要）

# 2. 构建
pnpm build

# 3. 检查发布内容（dry-run）
npm pack --dry-run
# 确认只有 dist/ 和 package.json，无源码泄露

# 4. 发布
npm publish
# 首次发布不需要 --access public（非 scope 包默认 public）
# 如果用 @scope/name 格式，需要加 --access public

# 5. 验证
npm info zcli-ai
# 或打开 https://www.npmjs.com/package/zcli-ai
```

### 发布后用户安装方式

```bash
# 方式 1：npx 直接运行（不安装）
npx zcli-ai

# 方式 2：全局安装
npm install -g zcli-ai
zcli

# 方式 3：项目内安装
npm install zcli-ai
npx zcli
```

## 四、后续版本更新流程

```bash
# 1. 改代码 + 测试通过

# 2. 更新版本号
npm version patch   # 0.0.1 → 0.0.2
# 或
npm version minor   # 0.0.1 → 0.1.0

# 3. 构建 + 发布
pnpm build && npm publish

# 4. 推送 tag
git push origin main --tags
```

## 五、可选优化

### README.md

npm 包页面会展示 README.md。当前项目根目录没有 README，建议创建一个包含：
- 项目介绍 + 截图/GIF
- 安装方式
- 快速开始
- 支持的模型列表
- 功能特性
- 配置说明

### prepublishOnly 脚本

自动在 publish 前构建 + 测试：

```json
"scripts": {
  "prepublishOnly": "pnpm test && pnpm build"
}
```

### 发布到 GitHub Packages（可选）

除了 npm，也可以发布到 GitHub Packages：

```bash
npm publish --registry=https://npm.pkg.github.com
```

需要在 package.json 里加：
```json
"publishConfig": {
  "registry": "https://npm.pkg.github.com"
}
```
