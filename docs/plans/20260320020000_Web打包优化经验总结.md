# Web 打包优化经验总结

> 日期: 2026-03-20
> 状态: 已完成
> 技术栈: Vite 6 + React 18 + Tailwind + TypeScript

---

## 一、优化前问题

Web Dashboard 单文件打包产物 **956 KB**（gzip 289 KB），全部代码打在一个 `index-*.js` 里。

用户打开任何页面（包括只看 ChatPage）都要下载整个 956 KB，其中大部分是当前页面用不到的依赖。

```
优化前：
  dist/assets/index-*.js     955.58 KB  ← 单文件，所有依赖 + 所有页面
  dist/assets/index-*.css     38.94 KB
  dist/index.html              0.45 KB
```

---

## 二、依赖体积分析

| 依赖 | 估计体积 | 使用范围 | 问题 |
|------|----------|----------|------|
| **recharts** | ~400 KB | 仅 OverviewPage（1 个页面） | 首屏不需要 |
| **highlight.js** | ~200 KB | 仅 MessageBubble 代码高亮 | 全量语言包，实际只用十几种 |
| **react-markdown + remark-gfm + rehype-highlight** | ~150 KB | 仅 MessageBubble | 首屏可能需要 |
| **react-router-dom** | ~50 KB | 全局路由 | 必要，无法优化 |
| **react + react-dom** | ~130 KB | 核心 | 必要，无法优化 |

**结论**：recharts 和 highlight.js 占了 600+ KB，但只在特定页面/组件使用。

## 三、优化手段

### 3.1 路由懒加载（React.lazy + Suspense）

**原理**：非首屏页面不在主 bundle 中，用户导航到时才按需下载。

```typescript
// 优化前：静态 import，全部打入主 bundle
import { OverviewPage } from './pages/OverviewPage'
import { ConversationsPage } from './pages/ConversationsPage'
import { SettingsPage } from './pages/SettingsPage'

// 优化后：lazy import，独立 chunk，按需加载
const OverviewPage = lazy(() =>
  import('./pages/OverviewPage').then(m => ({ default: m.OverviewPage }))
)
const ConversationsPage = lazy(() =>
  import('./pages/ConversationsPage').then(m => ({ default: m.ConversationsPage }))
)
```

**注意**：`lazy()` 需要 default export。如果组件是 named export，用 `.then(m => ({ default: m.XXX }))` 适配。

**Suspense 包裹**：
```tsx
<Suspense fallback={<PageLoading />}>
  <Routes>
    <Route path="/overview" element={<OverviewPage />} />
    ...
  </Routes>
</Suspense>
```

**效果**：OverviewPage（含 recharts）从首屏移出，变成 `OverviewPage-*.js` 独立 chunk。

### 3.2 Vite manualChunks 代码分割

**原理**：将第三方依赖按使用频率和更新频率分组，独立打包。浏览器可以分别缓存，依赖不变时无需重新下载。

```typescript
// vite.config.ts
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        // React 核心 — 几乎不变，长缓存命中率高
        'vendor-react': ['react', 'react-dom', 'react-router-dom'],
        // Markdown 渲染 — 只在消息气泡用
        'vendor-markdown': ['react-markdown', 'remark-gfm', 'rehype-highlight'],
        // 图表 — 只在概览页用，配合路由懒加载效果最大
        'vendor-charts': ['recharts'],
      },
    },
  },
}
```

**分包策略原则**：
- **高频使用 + 低变更**：独立包（如 vendor-react），浏览器长缓存
- **低频使用 + 大体积**：独立包 + 路由懒加载（如 vendor-charts）
- **业务代码**：不手动分包，让 Vite 自动 code-split

### 3.3 highlight.js 按需注册语言

**原理**：`highlight.js` 默认加载 190+ 种语言（~200KB），但编程 CLI 助手实际只需十几种常用语言。

```typescript
// 优化前：rehype-highlight 默认引入全量 highlight.js
import rehypeHighlight from 'rehype-highlight'
// → 打包时包含所有 190+ 语言定义

// 优化后：手动注册需要的语言
import hljs from 'highlight.js/lib/core'
import javascript from 'highlight.js/lib/languages/javascript'
import typescript from 'highlight.js/lib/languages/typescript'
import python from 'highlight.js/lib/languages/python'
import bash from 'highlight.js/lib/languages/bash'
import json from 'highlight.js/lib/languages/json'
// ... 共 12 种常用语言

hljs.registerLanguage('javascript', javascript)
hljs.registerLanguage('typescript', typescript)
// ...
```

**注册的语言列表**：
javascript, typescript, python, bash, json, css, xml/html, java, go, sql, yaml, markdown

### 3.4 CSS @import 顺序修复

**问题**：PostCSS 规范要求 `@import` 必须在所有其他语句之前（除 `@charset`）。

```css
/* 错误：@import 在 @tailwind 之后 */
@tailwind base;
@tailwind components;
@tailwind utilities;
@import 'highlight.js/styles/github-dark.css';  /* ⚠️ 警告 */

/* 正确：@import 在最前 */
@import 'highlight.js/styles/github-dark.css';
@tailwind base;
@tailwind components;
@tailwind utilities;
```

## 四、优化结果

### 打包产物对比

| 指标 | 优化前 | 优化后 | 改善 |
|------|--------|--------|------|
| **总 JS 文件数** | 1 个 | 8 个 | 代码分割 |
| **总 JS 体积** | 956 KB | 956 KB | 总量不变（只是分包） |
| **首屏 JS** | 956 KB | **207 KB** | **-78%** |
| **CSS 警告** | 1 个 | 0 个 | 已修复 |

### 首屏加载路径

```
用户打开 ChatPage（最常用场景）：
  index.js           28 KB   ← 核心业务代码
  vendor-react.js   179 KB   ← React 生态
  index.css          40 KB   ← 样式
  ─────────────────────────
  首屏总计:         247 KB   ← 比优化前减少 74%

用户导航到 OverviewPage（按需加载）：
  OverviewPage.js     9 KB   ← 页面代码
  vendor-charts.js  373 KB   ← recharts（此时才下载）

用户查看带代码高亮的消息（按需加载）：
  vendor-markdown.js 336 KB  ← react-markdown + rehype-highlight
```

### 各 chunk 详情

```
dist/assets/
├── index-*.js              28 KB   ← 核心：ChatPage + hooks + WebSocket
├── vendor-react-*.js      179 KB   ← React + ReactDOM + Router（长缓存）
├── vendor-markdown-*.js   336 KB   ← Markdown 渲染（消息气泡用时加载）
├── vendor-charts-*.js     373 KB   ← recharts（OverviewPage 懒加载）
├── SettingsPage-*.js       24 KB   ← 设置页
├── OverviewPage-*.js        9 KB   ← 概览页
├── ConversationsPage-*.js   6 KB   ← 回放页
├── LogsPage-*.js            1 KB   ← 日志页
├── useApi-*.js              1 KB   ← 共享 API hook
└── index-*.css             40 KB   ← 全局样式
```

## 六、关键原则总结

1. **首屏只加载必要代码** — 路由懒加载是最有效的手段
2. **大依赖独立分包** — manualChunks 让浏览器缓存生效
3. **按需加载语言/功能** — highlight.js 按需注册，不要全量引入
4. **CSS 规范遵守** — `@import` 必须在最前面
5. **分析先于优化** — 先用 `vite-bundle-visualizer` 看清谁占了空间
