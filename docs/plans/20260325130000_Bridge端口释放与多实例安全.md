# Bridge Server 端口释放与多实例安全

> 日期: 2026-03-25
> 状态: 已完成

---

## 一、问题

CLI 非正常退出后（kill -9、崩溃、窗口直接关闭），Web 页面无法打开。

**根因**：
1. `exitGracefully()` 里没有调用 `closeBridge()` 关闭 HTTP server
2. `process.on('exit', disconnectBridge)` 只断开 WS 连接，不释放端口
3. 端口 9800 残留在 TIME_WAIT 状态（Windows 默认约 120 秒）
4. 下次启动绑定 9800 失败 → Bridge Server 没启动 → Web 打不开

---

## 二、多实例场景分析

CCode 支持多 CLI 实例共享一个 Bridge Server：

```
CLI-1（owner）：启动 Bridge Server（端口 9800）+ 连接 WS
CLI-2（非 owner）：检测到 9800 被占 → 只连接 WS，不启动 server
CLI-3（非 owner）：同上
```

**关键约束**：只有 owner 退出时才能关 server，非 owner 退出不能影响其他实例。

---

## 三、改动

### 3.1 正常退出：owner 关 server（`bin/ccli.ts`）

```typescript
let isBridgeOwner = false  // 标记当前实例是否是 server 启动者

if (!portInUse) {
  const bridge = startBridgeServer({ dev: isDevMode })
  bridgePort = bridge.port
  isBridgeOwner = true
}

process.on('exit', () => {
  disconnectBridge()                        // 所有实例都断 WS
  if (isBridgeOwner) closeBridge()          // 只有 owner 关 server
})
```

### 3.2 非正常退出：端口自动递增（`src/server/bridge/server.ts`）

owner 被 kill -9 时 `process.on('exit')` 不会触发，端口残留在 TIME_WAIT。
下次启动时自动尝试下一个端口：

```typescript
const MAX_PORT_RETRIES = 3

let bindPort = port  // 默认 9800
for (let retry = 0; retry <= MAX_PORT_RETRIES; retry++) {
  try {
    server = serve({ fetch: app.fetch, port: bindPort })
    break
  } catch (err) {
    if (isEADDRINUSE(err) && retry < MAX_PORT_RETRIES) {
      bindPort++  // 9800 → 9801 → 9802 → 9803
    } else {
      throw err
    }
  }
}
```

### 3.3 `closeBridge` 导出

从 `server.ts` 私有函数改为导出，通过 `index.ts` 暴露给 `ccli.ts`。

## 四、行为矩阵

| 场景 | 行为 | 端口 |
|------|------|------|
| 正常退出（Ctrl+C / exit） | owner 关 server，释放端口 | 立即可用 |
| owner 被 kill -9 | 端口残留 TIME_WAIT | 下次自动递增到 9801 |
| 非 owner 退出 | 只断 WS，不影响 server | 不变 |
| 多个实例同时非正常退出 | 多个端口残留 | 下次从 9800 开始逐个尝试 |
| 递增到 MAX_PORT_RETRIES | 抛出异常，启动失败 | — |

---

## 五、遗留

- **Web URL 硬编码 9800**：`App.tsx` 中显示的 `http://localhost:9800/session/...` 仍然写死，
  当端口递增时显示不准。后续需要把实际端口通过 props 或环境变量传入 App 组件。

- **TIME_WAIT 上限**：Windows 默认 120 秒，`MAX_PORT_RETRIES = 3` 可覆盖同时残留 3 个端口的极端场景。
  如果未来需要更多，调大常量即可。
