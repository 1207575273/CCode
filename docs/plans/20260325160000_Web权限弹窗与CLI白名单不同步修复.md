# Web 权限弹窗与 CLI 白名单不同步修复

> 日期: 2026-03-25
> 状态: 已完成

---

## 一、问题现象

用户在 `settings.local.json` 中配置了全量工具放行（`Bash(*)`、`Write(*)`、`Edit(*)` 等），
CLI 端执行工具时确实不弹权限确认，直接开干。

但 Web 端仍然弹出权限确认弹窗（PermissionCard），且一直挂着直到 CLI 执行完工具后才消失。

**用户体验**：CLI 已经在干活了，Web 上还卡着一个无意义的"是否允许 Write？"弹窗。

---

## 二、根因分析

### 事件广播流程（修复前）

```
AgentLoop yield permission_request
  ↓
useChat 收到事件
  ↓
第 311 行：eventBus.emit(event)      ← ❌ 无条件广播！Web 立刻收到弹窗
  ↓
第 400-408 行：检查白名单
  ├─ permissionManager.isAllowed('write_file') → true
  └─ event.resolve(true) → 工具直接执行

Web 端：
  收到 permission_request → 显示 PermissionCard → 等待用户响应
  但 CLI 已经 resolve 了 → 工具执行完毕 → 后续事件到来 → 弹窗才消失
```

**核心问题**：`eventBus.emit(event)` 在权限检查**之前**执行，所有 `permission_request` 事件无差别广播给 Web，
即使 CLI 端会自动放行（白名单命中），Web 也会收到并弹窗。

### 权限检查的实际生效点

权限检查发生在 `useChat.ts`（UI 层），不是在 `agent-loop.ts`（引擎层）：

```
agent-loop.ts #checkPermission():
  - 只检查 isSidechain 和 isDangerous
  - dangerous=true 就 yield permission_request（不管白名单）

useChat.ts 事件处理:
  - 收到 permission_request 后才检查白名单
  - 匹配 → resolve(true)
  - 不匹配 → 弹窗
```

所以 `permission_request` 事件**一定会被 yield**，问题在于广播时机。

---

## 三、解决方案

### 核心改动：白名单命中时不广播 permission_request

**文件**：`src/ui/useChat.ts`（1 处，约 5 行）

```typescript
// 修复前：无条件广播
eventBus.emit(event)

// 修复后：跳过会被白名单自动放行的 permission_request
const skipBroadcast = event.type === 'permission_request' && (
  permissionManagerRef.current?.isAllowed(event.toolName) ||
  allowedToolsRef.current.has(event.toolName)
)
if (!skipBroadcast) eventBus.emit(event)
```

### 修复后的流程

```
AgentLoop yield permission_request
  ↓
useChat 收到事件
  ↓
检查白名单：Write(*) 匹配？
  ├─ 是 → skipBroadcast=true
  │   ├─ 不 emit 到 EventBus → Web 不收到 → 无弹窗 ✓
  │   └─ resolve(true) → 工具直接执行
  │
  └─ 否 → skipBroadcast=false
      ├─ emit 到 EventBus → Web 收到 → 弹窗询问
      └─ setPendingPermission → CLI 也弹窗
```

## 四、CCode 权限机制全景

### 四层检查优先级

```
1. isSidechain（子 Agent）     → 自动放行（agent-loop.ts）
2. isDangerous === false        → 自动放行（agent-loop.ts）
3. 项目级白名单 settings.local.json → 放行，不通知 Web（useChat.ts）
4. Session 级白名单（用户选过 "always"）→ 放行，不通知 Web（useChat.ts）
5. 弹窗询问                    → CLI PermissionDialog + Web PermissionCard
```

### CLI vs Web 权限处理对比

| 维度 | CLI 模式 | Web 模式 |
|------|----------|----------|
| 项目级白名单 | ✅ 直接读取 settings.local.json | ❌ 无法读取本地文件（通过 CLI 代理） |
| Session 级白名单 | ✅ allowedToolsRef 内存保存 | ✅ "始终允许"传 always=true → CLI 加入白名单 |
| 弹窗选项 | 3 个：Yes / Yes,always / No | 3 个：允许 / 始终允许 / 拒绝 |
| 白名单命中时 | 直接放行，**不广播给 Web** | 不会收到事件 |
| 白名单不命中时 | CLI 弹窗 + 转发给 Web | Web 弹窗，响应传回 CLI |

### settings.local.json 当前配置

```json
// 项目/.ccode/settings.local.json
{
  "permissions": {
    "allow": [
      "Bash(*)",        // → bash
      "Read(*)",        // → read_file
      "Write(*)",       // → write_file
      "Edit(*)",        // → edit_file
      "Glob(*)",        // → glob
      "Grep(*)",        // → grep
      "WebFetch(*)",    // → web_fetch
      "WebSearch(*)",   // → web_search
      "Task(*)",        // → task_output 等
      "NotebookEdit(*)",
      "LSP(*)",
      "mcp__*"          // 全部 MCP 工具
    ]
  }
}
```

**配置解析规则**（`src/config/permissions.ts`）：
- `Bash(*)` → 友好名解析 → 匹配 `bash` 工具
- `Write(*)` → 友好名 `write` + `_` 前缀匹配 → 匹配 `write_file`
- `mcp__*` → 前缀匹配所有 MCP 工具

## 五、Web "始终允许"按钮（已完成）

### 问题

CLI 权限弹窗有 3 个选项（Yes / Yes,always / No），Web 只有 2 个（允许 / 拒绝）。
Web 用户选"允许"后，下次遇到同一个工具还会再问，因为没有 always 标志。

### 修复

完整信号链路：

```
Web PermissionCard
  ├─ "允许"      → send({ type: 'permission', allow: true, always: false })
  ├─ "始终允许"  → send({ type: 'permission', allow: true, always: true })   ← 新增
  └─ "拒绝"      → send({ type: 'permission', allow: false, always: false })
        ↓
Bridge Server 转发
        ↓
bridge/client.ts
  → eventBus.emit({ type: 'permission_response', allow, always, source: 'web' })
        ↓
useChat.ts
  → resolvePermission(event.allow, event.always ?? false)
        ↓
always=true → allowedToolsRef.current.add(toolName) → 本会话后续不再询问
```

### 改动文件

| 文件 | 改动 |
|------|------|
| `web/src/components/PermissionCard.tsx` | 新增"始终允许"蓝色按钮（onAlwaysAllow prop） |
| `web/src/pages/ChatPage.tsx` | 传递 always 标志给 send() |
| `src/server/bridge/client.ts` | 透传 `always` 字段到 eventBus |
| `src/core/event-bus.ts` | `permission_response` 类型新增 `always?: boolean` |
| `src/ui/useChat.ts` | 接收 `event.always` 传给 `resolvePermission` |
