## 二、E2E 测试方案

### 2.1 现有集成测试（已实现）

```bash
# 需要配置 ~/.ccode/config.json 的 memory.embedding
pnpm vitest run tests/integration/memory-e2e.test.ts
```

**配置要求**：
```json
{
  "memory": {
    "enabled": true,
    "embedding": {
      "provider": "glm",
      "baseURL": "https://open.bigmodel.cn/api/paas/v4",
      "model": "embedding-3",
      "dimension": 1024
    }
  }
}
```

未配置时自动跳过，不阻塞 CI。

### 2.2 手动 E2E 验证流程

```
1. pnpm dev 启动项目
2. 对话中触发 memory_write（让 LLM 写几条记忆）
3. /remember list          — 验证记忆列表
4. /remember search 关键词  — 验证检索
5. /remember delete <id>   — 验证删除
6. 检查 .ccode/memory/ 目录下的 Markdown 文件
7. 用 SQLite 工具查看 ccode.db 的 memory_vectors 表
```

### 2.3 自动化 E2E 方案（待讨论）

**方案 A：脚本驱动**
- 写一个 `tests/e2e/memory-cli.sh` 脚本
- 通过 pipe 模式向 CLI 发送指令：`echo "/remember write ..." | pnpm dev`
- 优点：简单直接
- 缺点：依赖 LLM 响应，不确定性高

**方案 B：绕过 LLM 的 API 测试**
- 直接实例化 MemoryManager，模拟完整生命周期
- 现有 `memory-e2e.test.ts` 已经是这个思路
- 优点：确定性高，速度快
- 缺点：不覆盖 CLI 交互层

**方案 C：录制回放**
- 录制一次 LLM 交互的请求/响应
- 用 mock server 回放，测试 CLI 端到端
- 优点：确定性 + 覆盖全链路
- 缺点：录制维护成本

**初步倾向**：B 为主 + 手动验证补充，C 等记忆系统稳定后再考虑。

## 三、向量数据库可视化方案

### 3.1 数据位置

- 数据库文件：`~/.ccode/ccode.db`（libsql / SQLite 兼容）
- 向量表：`memory_vectors`
- 关键字段：`id`, `entry_id`, `scope`, `embedding`(F32_BLOB), `chunk_text`, `type`, `tags`

### 3.2 表结构查看工具

| 工具 | 平台 | 免费 | 特点 |
|------|------|------|------|
| **DB Browser for SQLite** | Win/Mac/Linux | 是 | 最常用的 SQLite GUI，能看表结构和数据 |
| **TablePlus** | Win/Mac/Linux | 部分 | 颜值高，多数据库支持 |
| **SQLite CLI** | 全平台 | 是 | 命令行快速验证 |
| **Beekeeper Studio** | Win/Mac/Linux | 社区版 | 现代 UI，支持 SQLite |

**快速验证命令**：
```bash
# 记录数
sqlite3 ~/.ccode/ccode.db "SELECT COUNT(*) FROM memory_vectors"

# 摘要
sqlite3 ~/.ccode/ccode.db "SELECT entry_id, scope, type, substr(chunk_text, 1, 60) FROM memory_vectors"

# 按 entry 聚合
sqlite3 ~/.ccode/ccode.db "SELECT entry_id, COUNT(*) as chunks FROM memory_vectors GROUP BY entry_id"
```

### 3.3 向量可视化工具（Embedding 质量分析）

目的：将高维向量降维到 2D/3D，观察聚类效果、发现离群点。

| 工具 | 方式 | 降维算法 | 适合场景 |
|------|------|---------|---------|
| **TensorFlow Embedding Projector** | 在线/本地 | t-SNE / PCA / UMAP | 快速查看，上传 TSV 即可 |
| **Nomic Atlas** | SaaS | UMAP | 大规模 embedding 可视化，自动聚类标注 |
| **Python + sklearn** | 本地脚本 | t-SNE / UMAP + matplotlib | 最灵活，可定制 |
| **Renumics Spotlight** | 本地 | UMAP | 专为 ML 数据探索设计，支持 DataFrame |
