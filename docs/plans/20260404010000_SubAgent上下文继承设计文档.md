## 一、问题全景

本次调研发现 cCli 的 AgentLoop 消息格式存在**三个根本性缺陷**，SubAgent 上下文缺失只是其中一个表现症状。

### 1.1 缺陷一：history 中没有 assistant 消息

`agent-loop.ts` 的 `#callLLM()` 返回后，LLM 的回复（text + tool_calls）**从未被追加到 history**：

```typescript
// agent-loop.ts:163-172
for (let turn = 0; turn < maxTurns; turn++) {
  const llmResult = yield* this.#callLLM(history)
  // ❌ llmResult 中的 text 和 toolCalls 没有 push 到 history
  yield* this.#executeToolCalls(llmResult.toolCalls, history)
  // 工具结果以 role:'user' 字符串追加
}
```

**后果**：多轮工具调用时，LLM 在第 N 轮**看不到自己第 N-1 轮说了什么**，只能看到工具结果。

### 1.2 缺陷二：工具结果用纯字符串而非结构化格式

```typescript
// agent-loop.ts:394-398 — 当前实现
history.push({
  role: 'user',
  content: `[Tool ${tc.toolName} result]: ${result.output}`,  // ← 纯字符串
})
```

**Anthropic 要求的格式**：
```json
{ "role": "user", "content": [
  { "type": "tool_result", "tool_use_id": "toolu_xxx", "content": "结果..." }
]}
```

**OpenAI 要求的格式**：
```json
{ "role": "tool", "tool_call_id": "call_xxx", "content": "结果..." }
```

cCli 两种都没用，而是用了自创的纯文本格式 `[Tool xxx result]: ...`。

### 1.3 缺陷三：连续 user 消息违反 Anthropic 交替规则

当前 history 的实际结构：

```
[user] "用户消息"                        ← 原始请求
[user] "[Tool bash result]: ls output"   ← 工具结果（字符串）
[user] "[Tool read_file result]: ..."    ← 工具结果（字符串）
[user] "[Tool bash result]: ..."         ← 工具结果（字符串）
```

**Anthropic API 要求严格 user/assistant 交替**，连续 user 消息直接返回 400 错误。

### 1.4 为什么现在能工作

**因为主力用的是 GLM（OpenAI 兼容格式）**——OpenAI API 不强制 user/assistant 交替，连续 user 消息不报错。加上 `toLangChainMessages()` 将所有内容提取为纯文本，掩盖了结构化信息的缺失。

但一旦切到 Anthropic 做主力且在 agent-loop 内多轮工具调用，就会出问题。

### 1.5 SubAgent 上下文缺失

在这三个缺陷的基础上，SubAgent 还有第四个问题：

```typescript
// dispatch-agent.ts:229
subLoop.run([{ role: 'user', content: prompt }])
// ← 只有一条消息，完全没有主 Agent 的历史
```

## 二、API 消息格式规范（调研结果）

### 2.1 Anthropic Messages API

```
完整的工具调用轮次：

user: "帮我查天气"
  ↓
assistant: {
  content: [
    { type: "text", text: "我来查一下" },
    { type: "tool_use", id: "toolu_1", name: "get_weather", input: { city: "北京" } }
  ]
}
  ↓
user: {
  content: [
    { type: "tool_result", tool_use_id: "toolu_1", content: "25°C 晴" }
  ]
}
  ↓
assistant: "北京现在 25°C，晴天。"
```

**规则**：
- role 必须 user/assistant **严格交替**，违反 = 400 错误
- assistant 可包含多个 tool_use（并行调用），对应 user 消息中必须有全部 tool_result
- tool_result.tool_use_id 必须匹配前面 assistant 中的 tool_use.id

### 2.2 OpenAI Chat Completions API

```
完整的工具调用轮次：

user: "帮我查天气"
  ↓
assistant: {
  content: null,
  tool_calls: [
    { id: "call_1", type: "function", function: { name: "get_weather", arguments: '{"city":"北京"}' } }
  ]
}
  ↓
tool: { role: "tool", tool_call_id: "call_1", content: "25°C 晴" }
  ↓
assistant: "北京现在 25°C，晴天。"
```

**规则**：
- **不强制** user/assistant 交替（tool 消息可以连续出现）
- tool_calls 是 assistant 消息的顶层字段（不在 content 里）
- arguments 是 JSON **字符串**（不是对象）
- 每个 tool_call 对应一条独立的 `role:"tool"` 消息

### 2.3 LangChain 统一抽象

```
HumanMessage → AIMessage(tool_calls=[]) → ToolMessage(tool_call_id=) → AIMessage
```

LangChain 在 adapter 层做转换：
- Anthropic：ToolMessage → user + tool_result blocks，自动合并连续消息
- OpenAI：ToolMessage → role:"tool" 独立消息

### 2.4 关键对比

| 维度 | Anthropic | OpenAI |
|------|-----------|--------|
| tool_use 位置 | assistant.content[] block | assistant.tool_calls[] 顶层字段 |
| tool_result 位置 | user.content[] block | 独立 role:"tool" 消息 |
| 多工具结果 | 合并到同一条 user 消息 | 每个一条独立 tool 消息 |
| role 交替 | **严格交替**（违反 = 400） | **不强制** |
| arguments | 原生对象 | JSON 字符串 |

---

## 三、设计目标

1. **消息格式标准化**：history 中正确包含 assistant 消息（text + tool_calls）和结构化工具结果
2. **双协议兼容**：同一份 history 能正确转换为 Anthropic 和 OpenAI 格式
3. **SubAgent 上下文继承**：子 Agent 能看到主 Agent 的对话历史
4. **向后兼容**：不破坏现有 UI 层（useChat.ts）的事件消费逻辑

## 四、方案设计

### 4.1 改造一：AgentLoop 追加 assistant 消息到 history

这是最核心的改动。`#callLLM()` 返回时需要同时返回累积的文本和工具调用，然后在 `run()` 主循环中追加到 history。

```typescript
// agent-loop.ts — #callLLM() 的返回类型扩展
interface LLMResult {
  toolCalls: ToolCallContent[]
  text: string          // 新增：累积的文本
  aborted: boolean
}

// #callLLM() 中累积文本
async *#callLLM(history: Message[]): AsyncGenerator<AgentEvent, LLMResult> {
  let accumulatedText = ''
  const pendingToolCalls: ToolCallContent[] = []

  for await (const chunk of this.#provider.chat(chatRequest)) {
    const mapped = this.#mapChunk(chunk, pendingToolCalls)
    if (mapped) {
      if (mapped.type === 'text') accumulatedText += mapped.text  // 累积文本
      yield mapped
    }
    // ... usage / done 处理不变
  }

  return { toolCalls: pendingToolCalls, text: accumulatedText, aborted: false }
}

// run() 主循环中追加 assistant 消息
async *run(messages: Message[]): AsyncIterable<AgentEvent> {
  const history: Message[] = [...messages]

  for (let turn = 0; turn < maxTurns; turn++) {
    const llmResult = yield* this.#callLLM(history)
    if (llmResult.aborted) return

    // ✅ 追加 assistant 消息到 history（text + tool_calls）
    const assistantContent: MessageContent[] = []
    if (llmResult.text) {
      assistantContent.push({ type: 'text', text: llmResult.text })
    }
    for (const tc of llmResult.toolCalls) {
      assistantContent.push(tc)  // ToolCallContent 本身就是 MessageContent
    }
    if (assistantContent.length > 0) {
      history.push({ role: 'assistant', content: assistantContent })
    }

    if (llmResult.toolCalls.length === 0) {
      yield { type: 'done', reason: 'complete' }
      return
    }

    yield* this.#executeToolCalls(llmResult.toolCalls, history)
  }
}
```

### 4.2 改造二：工具结果用结构化格式

将工具结果从纯字符串改为 `ToolResultContent`：

```typescript
// agent-loop.ts — #executeOneTool() 中
// 改前：
history.push({
  role: 'user',
  content: `[Tool ${tc.toolName} result]: ${result.output}`,
})

// 改后：
history.push({
  role: 'user',
  content: [{
    type: 'tool_result',
    toolCallId: tc.toolCallId,
    result: result.output,
    ...(result.success === false ? { isError: true } : {}),
  }],
})
```

并行执行路径（`#executeToolCalls` 中的 safe tools）同理，多个工具结果合并到**同一条** user 消息：

```typescript
// 并行工具结果合并为一条 user 消息（Anthropic 要求同一条消息包含所有 tool_result）
if (safe.length > 0) {
  // ... 并行执行

    type: 'tool_result' as const,
    toolCallId: pr.toolCallId,
    result: pr.success ? pr.output : (pr.error ?? 'error'),
    ...(pr.success === false ? { isError: true } : {}),
```

### 4.3 改造三：toAnthropicMessages() 适配结构化内容

当前的 `toAnthropicMessages()` 已经有处理 `tool_call` 和 `tool_result` 类型的代码（`anthropic.ts:36-64`），只是从未被触发过（因为 history 中没有这些类型）。

需要验证并修正几个点：

```typescript
function toAnthropicMessages(messages: Message[]): Anthropic.MessageParam[] {
  const result: Anthropic.MessageParam[] = []

  for (const msg of messages) {
    if (msg.role === 'system') continue

    // ... 现有的 string / array 处理

    // 关键修正：tool_result 不再作为独立 user 消息 push，
    // 而是作为当前 user 消息的 content block
    // （因为改造二已将多个 tool_result 合并到一条 user 消息中）
  }

  // 后处理：合并连续的 user 消息（防御性，理论上改造后不会出现）
  return mergeConsecutiveRoles(result)
}

/** 合并连续相同 role 的消息（Anthropic 交替规则防御） */
function mergeConsecutiveRoles(messages: Anthropic.MessageParam[]): Anthropic.MessageParam[] {
  const merged: Anthropic.MessageParam[] = []
  for (const msg of messages) {
    const last = merged[merged.length - 1]
    if (last && last.role === msg.role) {
      // 合并 content
      const lastContent = Array.isArray(last.content) ? last.content : [{ type: 'text' as const, text: last.content }]
      const curContent = Array.isArray(msg.content) ? msg.content : [{ type: 'text' as const, text: msg.content }]
      last.content = [...lastContent, ...curContent]
    } else {
      merged.push({ ...msg })
    }
  }
  return merged
}
```

### 4.4 改造四：toLangChainMessages() 适配结构化内容

当前 `message-converter.ts` 的 `toLangChainMessages()` 只提取文本，丢失了 tool_call 结构。需要改造：

```typescript
// message-converter.ts — 改造后
import { HumanMessage, AIMessage, SystemMessage, ToolMessage } from '@langchain/core/messages'

export function toLangChainMessages(messages: Message[]): BaseMessage[] {
  const result: BaseMessage[] = []

  for (const msg of messages) {
    const blocks = normalizeContent(msg.content)

    if (msg.role === 'system') {
      result.push(new SystemMessage(extractText(blocks)))
      continue
    }

    if (msg.role === 'assistant') {
      const text = extractText(blocks)
      const toolCalls = blocks
        .filter((b): b is ToolCallContent => b.type === 'tool_call')
        .map(tc => ({
          id: tc.toolCallId,
          name: tc.toolName,
          args: tc.args,
        }))

      result.push(new AIMessage({
        content: text || '',
        ...(toolCalls.length > 0 ? { tool_calls: toolCalls } : {}),
      }))
      continue
    }

    if (msg.role === 'user') {
      // 检查是否包含 tool_result
      const toolResults = blocks.filter((b): b is ToolResultContent => b.type === 'tool_result')
      if (toolResults.length > 0) {
        // 每个 tool_result 转为独立 ToolMessage（LangChain / OpenAI 格式）
        for (const tr of toolResults) {
          result.push(new ToolMessage({
            content: typeof tr.result === 'string' ? tr.result : JSON.stringify(tr.result),
            tool_call_id: tr.toolCallId,
          }))
        }
      } else {
        result.push(new HumanMessage(extractText(blocks)))
      }
      continue
    }
  }

  return result
}

function normalizeContent(content: MessageContent | MessageContent[] | string): MessageContent[] {
  if (typeof content === 'string') return [{ type: 'text', text: content }]
  return Array.isArray(content) ? content : [content]
}
```

**trimHistoryForSubAgent** — 现在 history 有正确的 user/assistant 交替格式，裁剪时需要保持交替完整性：

```typescript
// src/tools/agent/context-utils.ts

export function trimHistoryForSubAgent(
  history: ReadonlyArray<Message>,
  policy?: ContextPolicy,
): Message[] {
  const { maxMessages = 20, maxTokenEstimate = 8000 } = policy ?? {}

  if (history.length === 0) return []

  // 1. 截断到最后一个完整轮次（assistant 的所有 tool_call 都有对应 tool_result）
  const clean = truncateToCompleteRound(history)

  // 2. 保留第一条 user（原始需求）
  const firstUser = clean.find(m => m.role === 'user')

  // 3. 从尾部取最近 maxMessages 条，保持 user/assistant 交替完整
  let recent = takeRecentCompleteRounds(clean, maxMessages)

  // 4. 如果 firstUser 不在 recent 中，插到开头
  if (firstUser && !recent.includes(firstUser)) {
    recent = [firstUser, ...recent]
  }

  // 5. token 预算裁剪
  while (estimateTokens(recent) > maxTokenEstimate && recent.length > 2) {
    // 移除最前面的完整轮次（user + assistant 对）
    if (recent.length >= 3 && recent[0] === firstUser) {
      recent.splice(1, 2)  // 保留 firstUser，移除紧跟的一对
    } else {
      recent.splice(0, 2)
    }
  }

  // 6. 截断超长工具结果
  return recent.map(truncateLongContent)
}

/** 截断到最后一个完整的工具调用轮次 */
function truncateToCompleteRound(history: ReadonlyArray<Message>): Message[] {
  const result = [...history]

  // 从后往前找最后一条 assistant 消息
  for (let i = result.length - 1; i >= 0; i--) {
    if (result[i].role !== 'assistant') continue

    const blocks = normalizeContent(result[i].content)
    const toolCalls = blocks.filter(b => b.type === 'tool_call')
    if (toolCalls.length === 0) break  // 纯文本 assistant，完整

    // 检查后续是否有所有 tool_call 的 tool_result
    const toolCallIds = new Set(toolCalls.map((tc: ToolCallContent) => tc.toolCallId))
    for (let j = i + 1; j < result.length; j++) {
      const jBlocks = normalizeContent(result[j].content)
      for (const b of jBlocks) {
        if (b.type === 'tool_result') toolCallIds.delete((b as ToolResultContent).toolCallId)
      }
    }

    if (toolCallIds.size > 0) {
      // 不完整，截断到这条 assistant 之前
      result.length = i
    }
    break
  }

  return result
}

/** 从尾部取最近的完整轮次 */
function takeRecentCompleteRounds(messages: Message[], maxCount: number): Message[] {
  if (messages.length <= maxCount) return [...messages]
  // 从尾部取 maxCount 条，然后确保第一条是 user（保持交替起点正确）
  let start = messages.length - maxCount
  while (start < messages.length && messages[start].role !== 'user') {
    start++
  }
  return messages.slice(start)
}
```

## 八、风险与缓解

| 风险 | 严重性 | 缓解 |
|------|--------|------|
| 改造后 Anthropic 和 OpenAI 行为不一致 | 高 | 双协议分别写测试用例，模拟完整的多轮工具调用场景 |
| 现有 UI 层依赖纯文本工具结果格式 | 中 | useChat.ts 消费的是 AgentEvent（tool_done 事件），不直接读 history，不受影响 |
| toAnthropicMessages 改动影响现有对话 | 中 | mergeConsecutiveRoles 做防御性合并，即使 history 格式异常也不报错 |
| SubAgent 收到的裁剪历史 token 估算不准 | 低 | maxTokenEstimate 是软限制，超一点不影响功能 |
| 并行工具结果合并影响事件顺序 | 低 | 事件 yield 顺序不变，只改 history 中的追加方式 |
