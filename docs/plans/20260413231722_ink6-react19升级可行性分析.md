## 一、升级目标版本矩阵

| 依赖 | 当前版本 | 升级目标 | 兼容性 |
|------|---------|---------|--------|
| **Node.js** | v20.19.3 | 不变 | ink 6 要求 >=20 ✅ |
| **react** | ^18.3.1 | ^19.2.0 | — |
| **ink** | ^5.2.1 | ^6.8.0 | — |
| **ink-spinner** | 5.0.0 | 5.0.0（不变） | peer ink>=4 ✅ |
| **ink-text-input** | 6.0.0 | 6.0.0（不变） | peer ink>=5 ✅ |

| **ink-testing-library** | 4.0.0 | 4.0.0（不变） | peer @types/react>=18 ✅ |

**结论：升级到 ink 6 可行。不建议跳到 ink 7（要求 Node>=22，当前 v20 不满足）。**

## 二、ink 5→6 Breaking Changes 分析

ink 6.0.0 的核心变化：

1. **Node.js 最低版本 >=20** — 当前 v20.19.3 满足
2. **React 最低版本 >=19** — 需同步升级
3. **无组件 API 删除** — Box、Text、Static、useInput、useStdin、useStdout 等 API 签名不变

### 关键行为变化（ink 6/7 中逐步引入）

| 变化点 | 影响 | 说明 |
|--------|------|------|
| ~~Backspace 统一走 `key.backspace`~~ | ~~**高**~~ → **无** | **实测确认：ink 6.8.0 的 `parse-keypress.js` 中 `\x7f` 仍然解析为 `key.delete`，与 ink 5 行为一致。** 源码注释明确：`// TODO: Merge them back together in the next major version.` 即 ink 7 才会改。InputBar 的 workaround 无需变动。 |
| Escape 不再附带 `key.meta` | **低** | ink 7 引入，ink 6 暂不受影响 |
| `react-devtools-core` 成为 peer dependency | **低** | 可选，不影响运行 |

## 三、react 18→19 Breaking Changes 分析

react 19 对函数组件 + Hooks 项目影响极小：

1. **无 class 组件** — cCli 全部使用函数组件 ✅
2. **useEffect cleanup 时序微调** — 需回归验证副作用清理是否有竞态
3. **无删除 API** — useState/useCallback/useRef/useMemo/useEffect 全部保留
4. **Concurrent 行为变化** — Ink 的渲染不依赖 React concurrent features，影响可忽略

### 关键发现

**ink 6 未修复 Linux Backspace 解析问题**（与升级前预估不同）：

```javascript
// ink 6.8.0 build/parse-keypress.js 第 431-434 行
else if (s === '\x7f' || s === '\x1b\x7f') {
    // TODO(vadimdemedes): `enquirer` detects delete key as backspace,
```

- InputBar 的 `key.backspace || key.delete` 双判断 workaround **必须保留**

## 七、后续 ink 7 升级备忘

ink 7.0.0 的额外变化（留待将来 Node 升级到 22+ 后处理）：

1. **Node.js >=22 必需**（当前 v20 不满足）
2. **Backspace 合并**：`\x7f` 将改为 `key.backspace`，届时需重构 InputBar
3. **Escape 行为**：单独按 Escape 不再设 `key.meta`（当前代码未依赖此行为，影响低）
4. **内部使用 `useEffectEvent`**：要求 react >=19.2.0（已满足）
