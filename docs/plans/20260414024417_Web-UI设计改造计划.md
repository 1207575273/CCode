# CCode Web UI 设计改造计划

> 目标：从"能用的开发者工具"提升到"有设计感的专业级产品"。
> 参考标杆：Linear（任务管理）、Raycast（效率工具）、Warp（终端）、Cursor（编辑器）。
> 原则：功能导向，克制装饰，信息密度优先，让界面"消失"让内容说话。
> 约束：不引入图标库（纯 SVG 内联 + Unicode 符号），支持黑白双主题。

### 1.2 问题清单

| 维度 | 问题 | 严重度 |
|------|------|--------|
| **导航** | emoji 图标(💬📊📋⚙️)不专业，辨识度差，跨平台渲染不一致 | 高 |

| **主题** | 纯深色模式，0 处 dark: 类，无亮色主题选项 | 高 |

| **图标** | 纯 emoji + 文字，视觉粗糙 | 高 |

### 1.3 做得好的地方（保留）

- 布局结构合理：左栏导航 + 右侧内容区，经典且有效
- Tailwind 使用规范：间距遵循 4px 步长，圆角统一 rounded-lg
- 代码块 + Markdown 渲染质量高（github-dark 主题 + prose 定制）
- WebSocket 实时架构可靠
- 阴影使用极度克制（仅 Toast 1 处 shadow-lg）
- 动画节制（仅 fade-in），没有花哨过渡

## 二、设计原则

### 2.1 核心理念

**"Terminal-native, not dashboard-cosplay"**

这是一个 CLI Agent 的 Web 伴侣界面，不是 SaaS 仪表板。用户是开发者，他们的审美标杆是 VS Code、Warp、Linear——不是 Notion 模板或 Dribbble 作品。

### 2.2 七条铁律

| # | 原则 | 释义 |
|---|------|------|
| 1 | **内容即界面** | 消息、代码、指标是主角。UI 框架要隐身，不要争夺注意力 |
| 2 | **单一主表面** | 每个页面只有一个核心区域占据视觉重心，不搞双栏竞争 |
| 3 | **层次靠间距和色阶** | 用 3 层背景色（base/surface/elevated）建立层次，不用阴影/渐变/边框堆砌 |
| 4 | **交互即时反馈** | 每个点击、悬停都有视觉响应，100-150ms，克制 |
| 5 | **信息渐进披露** | 不一次展示所有状态，hover/click 再展开细节 |
| 6 | **像素级一致** | 所有间距、圆角、字号、颜色从 Token 取，不允许 ad-hoc 值 |
| 7 | **双主题原生** | 黑白主题不是"反色"，而是两套独立调教的色板，都要好看 |

### 2.3 禁止清单

- 禁止 emoji 做图标
- 禁止引入图标库（Lucide/Heroicons 等），图标用内联 SVG 或 Unicode 符号
- 禁止 gradient 背景/文字
- 禁止 > 12px 圆角（8px 为主，4px 辅）
- 禁止 glassmorphism / 毛玻璃
- 禁止 transform 动画
- 禁止装饰性文案
- 禁止 KPI 卡片网格作为默认布局

## 四、双主题 Token 体系

### 4.1 主题切换机制

采用 Tailwind `darkMode: 'class'` 策略，通过 `<html class="dark">` 切换：

- `dark` 类存在 → 暗色主题
- `dark` 类不存在 → 亮色主题
- 用户偏好存入 `localStorage`，首次访问默认暗色主题（开发者工具的主流偏好）

### 4.6 主题切换 Hook

```typescript
// src/hooks/useTheme.ts
import { useState, useEffect } from 'react'

type Theme = 'light' | 'dark' | 'system'

export function useTheme() {
  const [theme, setTheme] = useState<Theme>(() => {
    return (localStorage.getItem('theme') as Theme) ?? 'system'
  })

  useEffect(() => {
    const root = document.documentElement
    const prefersDark = window.matchMedia('(prefers-color-scheme: dark)')

    function apply() {
      const isDark = theme === 'dark' || (theme === 'system' && prefersDark.matches)
      root.classList.toggle('dark', isDark)
    }

    apply()
    localStorage.setItem('theme', theme)
    prefersDark.addEventListener('change', apply)
    return () => prefersDark.removeEventListener('change', apply)
  }, [theme])

  return { theme, setTheme }
}
```

## 七、双主题视觉预期

### 暗色主题（Dark — Void Space）

```
底色 #0d1117（近纯黑）+ 面板 #161b22（微亮灰）+ 强调 #58a6ff（冷蓝）

```
底色 #ffffff（纯白）+ 面板 #f6f8fa（浅灰）+ 强调 #0969da（深蓝）
├─ 侧边栏: bg-surface，选中项左侧 2px accent 竖线
